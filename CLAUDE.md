# CLAUDE.md

Этот репозиторий — календарь пересекающихся рабочих графиков мамы (Наташа) и папы (Вова).

Нужен, чтобы заранее видеть:

- когда один из родителей будет дома и сможет сидеть с дочкой,
- когда оба на работе и придётся просить кого-то ещё понянчиться (тёща, договориться, брать с собой).

Подробности развёртывания — в [README.md](README.md).

---

## Что важно знать перед правками

### Источники данных
Оба календаря — публичные iCal-фиды Google. Скачиваются `urllib`'ом при каждом запуске `generate.py`:
- Наташа (`Смены Наташи В.`, TZ `Asia/Qyzylorda`) — все события одинаковые `SUMMARY: Работа`. Смены: будни 16:00–03:00, выходные 14:00–01:00 (~11 часов), ~раз в 4 дня.
- Вова (`Аренда КЦ ORTA`, TZ `Asia/Almaty`) — это календарь бронирований культурного центра ORTA, **каждое событие = рабочее обязательство**. Не путать с обычными встречами в Google Calendar.

Google iCal отдаёт скользящее окно событий в районе текущей даты — горизонт «уезжает» вперёд с каждым обновлением. Это не баг.

Display TZ — `Asia/Almaty` (UTC+5).

### Логика классификации (5 статусов)

`generate.py` выставляет «наивный» статус, основанный только на событиях. **Финальный статус считает фронт** в `effectiveStatus(day)` с учётом флага `availability[date]` из `localStorage`:

| Статус | Когда | Цвет |
|---|---|---|
| `red` | Наташа на смене + у Вовы есть событие пересекающееся со сменой и заканчивающееся **после 20:00** (KID_BEDTIME_HOUR), ИЛИ Наташа на смене + нет пересечений + папа НЕ отметил «Могу посидеть» | красный |
| `yellow` | Наташа на смене + есть пересечения с событиями Вовы, все заканчиваются ДО 20:00 | жёлтый |
| `green` | Наташа на смене + папа явно отметил «Могу посидеть» (взял отгул) | зелёный |
| `vova_only` | У Вовы есть события, у Наташи нет смены — Наташа дома с дочкой | голубой |
| `free` | У обоих ничего нет | серый |

Граница 20:00 — это `KID_BEDTIME_HOUR` в `generate.py`. Жёсткое правило: если папа на работе после этого времени — кто-то должен быть дома, потому что дочка уже не может ездить.

### Двухролевая система

Один и тот же payload шифруется ДВУМЯ паролями → два файла в `docs/`:
- `data.dad.enc.json` (пароль из ENV `SITE_PASSWORD_DAD`, дефолт `1133`)
- `data.mom.enc.json` (пароль из ENV `SITE_PASSWORD_MOM`, дефолт `0808`)

Фронт пробует расшифровать оба → роль определяется тем, какой подошёл. Роль хранится в `sessionStorage` и:
- **Папа** видит кнопку «Могу посидеть» в тултипе и может ставить отметки.
- **Мама** видит ту же таблицу, но без кнопки. Бейдж в шапке отличается (`👨 Папа` / `👩 Мама`).

### Что персонально для устройства, а что общее

| Хранится | Где | Видно другому устройству? |
|---|---|---|
| Календари + классификация | `docs/data.{role}.enc.json` (репо) | Да |
| Названия статусов (renamed labels) | `localStorage` ключ `tcrnv:labels` | Нет |
| Отметки «Могу посидеть» | `localStorage` ключ `tcrnv:availability` | **НЕТ — это сознательное решение пользователя**. На статическом GitHub Pages нет бэкенда, а пользователь явно отказался от внешнего хранилища (CF Workers и т.п.). Если папа отметил день у себя на телефоне — мама на своём этого не увидит. |
| Введённый пароль | `sessionStorage` `tcrnv:password` | Нет (per-tab) |
| Роль | `sessionStorage` `tcrnv:role` | Нет |

### Безопасность пароля
PBKDF2-HMAC-SHA256 (200 000 итераций) → AES-GCM. Это **обфускация, не криптозащита**: пароли короткие (4 цифры = 10к комбинаций), брутфорс возможен. Цель — отсеять случайного человека, нашедшего ссылку.

### Нетривиальные подводные камни (наступали)

1. **`[hidden]` атрибут перебивается `display: flex`**. UA stylesheet с `[hidden] { display: none }` слабее любого специфичного `display`. В `style.css` есть глобальный override `[hidden] { display: none !important; }` — НЕ удалять, иначе парольный экран не скрывается после успешного входа, и дашборд рисуется ПОД ним.

2. **Менеджеры паролей дописывают мусор** в поле `<input type="password">`. У поля паролей в `index.html` стоит букет атрибутов (`name="tcrnv-pin"`, `autocomplete="off"`, `data-1p-ignore`, `data-lpignore`, `data-bwignore`, `data-form-type="other"`) — без них Firefox/1Password подсовывают сохранённые значения и юзер видит «неверный пароль».

3. **Привязка обработчиков ДО async-операций**. В `init()` обработчик submit подключается синхронно ПЕРЕД попыткой авто-разблокировки из sessionStorage. Иначе пока фоном бежит PBKDF2 (~0.5–1с), кнопка «Войти» не реагирует и форма перезагружает страницу через дефолтный submit.

4. **Форма должна иметь `action="javascript:void(0)"`**. Безопасная сетка от перезагрузки страницы, если JS-обработчик когда-нибудь отвалится.

5. **Тултип с `pointer-events: none` блокирует кнопки внутри**. Когда тултип «зафиксирован» (`#tooltip.visible`), CSS поднимает `pointer-events: auto`.

6. **На Windows для `zoneinfo` нужен пакет `tzdata`**. У Python нет встроенной базы таймзон, поэтому в `requirements`-сетапе и в README-инструкции указано `pip install cryptography tzdata`.

### Развёртывание

GitHub Actions деплоит `docs/` в Pages. Workflow читает секреты `SITE_PASSWORD_DAD` и `SITE_PASSWORD_MOM`. Если секретов нет — берутся дефолты `1133` / `0808` из `generate.py`.

**Pages должен быть включён вручную** один раз: Settings → Pages → Source = «GitHub Actions». До этого деплой будет фейлиться 404 — это нормальный onboarding.
